mysql数据类型

May 21, 2011 | mysql | Hits

1 数字

1.1 整数

MySQL数据类型 含义
tinyint(m) 1个字节表示(-128~127)
smallint(m) 2个字节表示(-32768~32767)
mediumint(m) 3个字节表示(-8388608~8388607)
int(m) 4个字节表示(-2147483648~2147483647)
bigint(m) 8个字节表示(+-9.22*10的18次方)

1.2 浮点数

MySQL数据类型 含义
float(m,d) 单精度浮点型，8位精度(4字节),m是十进制数字的总个数，d是小数点后面的数字个数
double(m,d) 双精度浮点型，16位精度(8字节)

1.3 正点数

decimal(m,d) 定点类型浮点型在数据库中存放的是近似值，而定点类型在数据库中存放的是精确值。参数m是定点类型数字的最大个数（精度），范围为0~65，d小数点右侧数字的个数，范围为0~30，但不得超过m。对定点数的计算能精确到65位数字。

2 字符串

MySQL数据类型 含义
char(n) 固定长度的字符串，最多255个字节 ，根据编码不同，能够存储的字符数也不同。
varchar(n) 固定长度的字符串，最多65535个字节，根据编码不同，能够存储的字符数也不同。此类型根据设置最后的几个字节需要保存值的具体长度。如果声明为varchar（255），那么最后的一个字节用于存储长度，如果声明为varchar（256），那么因为一个字节已经无法保存，因此使用两个字节来保存长度。对于每个记录，mysql需要使用一个单独的字节来保存当前记录中的null值，如果想要节省此null值，需要记录中所有的列都为not null。
tinytext 可变长度字符串，最多255个字节
text 可变长度字符串，最多65535个字节
mediumtext 可变长度字符串，最多2的24次方-1个字节
longtext 可变长度字符串，最多2的32次方-1个字节

二进制数据(xxxBlob) XXXBLOB和xxxtext是对应的，不过存储方式不同，xxxTEXT是以文本方式存储的，如果存储英文的话区分大小写，而xxxBlob是以二进 制方式存储的，不区分大小写。xxxBlob存储的数据只能整体读出。xxxTEXT可以指定字符集，xxxblob不用指定字符集。

3 日期时间

timestamp比较特殊，如果定义一个字段的类型为timestamp，这个字段的时间会在其他字段修改的时候自动刷新。所以这个数据类型的字段可以存放这条记录最后被修改的时间，而不是真正来的存放时间。

类型名 描述
date 日期'2008-12-2'
time 时间'12:25:36'
datetime 日期时间'2008-12-2 22:06:44'
timestamp 不固定

4 数据类型的属性

MySQL关键字 含义
NULL 数据列可包含NULL值
NOT NULL 数据列不允许包含NULL值
DEFAULT xxx 默认值，如果插入记录的时候没有指定值，将取这个默认值
PRIMARY KEY 主键
AUTOINCREMENT 递增，如果插入记录的时候没有指定值，则在上一条记录的值上加1，仅适用于整数类型
UNSIGNED 无符号
CHARACTER SET name 指定一个字符集
